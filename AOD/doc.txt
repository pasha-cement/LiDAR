**Предложения по ПИВ**  
*Используемые сокращения:*  
- ПИВ – протокол информационного взаимодействия.  
- ДРВ – драйвер.  
- ЭВМ – электронная вычислительная машина. Вступает в роли мастера линии.  

### Формат сообщения UART  
#### Физический уровень  
На физическом уровне обмен осуществляется по шине стандарта USB. В ЭВМ устройство опознается как STM32 Virtual COM Port.  

#### Формат символа  
Интерфейс информационного обмена сконфигурирован следующим образом:  
- Скорость передачи данных: 115200 бод.  
- Каждый передаваемый символ содержит:  
  - нулевой стартовый бит;  
  - поле данных шириной 8 бит;  
  - бит четности;  
  - один стоповый бит.  

- Стартовый бит имеет фиксированное значение 0.  
- Поле данных представляет собой полезную нагрузку символа. Передача поля данных ведется в порядке от старшего бита к младшему.  
- Бит четности принимает такое значение, чтобы общее количество бит, равных 1, в совокупности поля данных и бита четности было четным.  
- Стоповый бит имеет фиксированное значение 1.  

*Далее по тексту оборот «значение символа» следует трактовать как значение поля данных символа.*  

#### Порядок информационного обмена  
Информационный обмен по интерфейсу ведется в порядке: запрос ведущего – ответ ведомого.  
ДРВ выступает в роли ведомого. Внешняя ЭВМ должна исполнять роль ведущего.

### Структура кадра  
Общая структура кадра информационного обмена представлена в таблице 3.2.  

**Таблица 3.2 – Структура кадра**  

| Наименование поля     | Стартовый символ | Ширина поля данных | Тип сообщения | Поле данных | Контрольная сумма |  
|-----------------------|------------------|--------------------|---------------|-------------|-------------------|  
| Размер, байт          | 1                | 1                  | 1             | 0…255       | 1                 |  
| Очередность передачи  | 1                | 2                  | 3             | 4           | 5                 |  

Кадр информационного обмена в общем случае состоит из 5 полей. Все поля, за исключением поля данных, являются обязательными. Поле данных может отсутствовать либо иметь ширину до 255 символов.  

- **Поле «Стартовый символ»** имеет ширину в 1 символ. Значение этого символа фиксировано и равно 0xAA.  
- **Поле «Ширина поля данных»** имеет ширину в 1 символ. Значение этого символа равно количеству символов в поле данных – от 0 до 255.  
- **Поле «Тип сообщения»** имеет ширину в 1 символ. В значении этого символа закодировано назначение кадра и информации, содержащейся в поле данных, при его наличии.  
- **Поле данных** должно иметь ширину, равную значению, содержащемуся в поле «Ширина поля данных». Значения символов поля данных, в том числе их семантическое содержание, определяется значением поля «Тип сообщения».  
- **Поле «Контрольная сумма»** имеет ширину в 1 символ. Значение этого символа равно контрольной сумме, рассчитанной по значениям всех символов кадра, начиная со стартового символа и заканчивая последним символом поля данных.  

Очередность передачи полей кадра отражена в таблице 3.2. Символы, составляющие один кадр, должны следовать друг за другом с интервалом, не превышающим длительности передачи одного символа.  

Структура кадра должна быть признана некорректной, а сам кадр должен быть отброшен принимающей стороной в следующих случаях:  
- стартовый символ содержит некорректное значение;  
- количество символов в поле данных не соответствует значению символа в поле «Ширина поля данных»;  
- символ поля «Тип сообщения» содержит значение, не оговоренное в соответствующем разделе настоящего описания;  
- значение символа, переданного в поле «Контрольная сумма», не соответствует значению контрольной суммы, вычисленной на основе значений остальных символов кадра.  

Также принятый кадр должен быть отброшен, если при приеме хотя бы одного из входящих в него символов будет обнаружено неверное значение бита четности.  
После каждой верно принятой команды ДРВ высылает сообщение-подтверждение о корректном получении команды.  

---

### Расчет контрольной суммы  
Контрольная сумма для каждого кадра рассчитывается по значениям всех символов кадра, начиная со стартового символа и заканчивая последним символом поля данных.  

Расчет контрольной суммы осуществляется по алгоритму получения 8-битного циклического избыточного кода (CRC-8) со следующими вводными условиями:  
- порождающий полином: 0x31 (CRC-8-Dallas/Maxim);  
- начальное значение регистра: 0xFF;  
- преинверсия отсутствует;  
- постинверсия отсутствует.  

На проверочной последовательности входных данных:  
`0x31 0x32 0x33 0x34 0x35 0x36 0x37 0x38 0x39`  
алгоритм должен давать результат `0xF7`.
